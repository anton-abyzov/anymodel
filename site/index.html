CLI Reference

anymodel [command] [options]

Commands:
  claude        Run Claude Code directly (no proxy)
  proxy         Start the proxy server
  gpt           Connect to proxy with GPT-4o
  gemini        Connect to proxy with Gemini 2.5 Flash
  qwen          Connect to proxy with Qwen3 Coder (free)
  llama         Connect to proxy with Llama 3.3 70B (free)
  deepseek      Connect to proxy with DeepSeek R1
  nemotron      Connect to proxy with Nemotron 120B (free)
  proxy ollama  Proxy with local Ollama

Options:
  --model, -m     Model ID (e.g., qwen/qwen3-coder:free)
  --port, -p      Proxy port (default: 9090)
  --free-only     Only allow free models
  --token, -t     Require auth token
  --rpm           Rate limit per minute (default: 60)
  --help, -h      Show help

Environment Variables

OPENROUTER_API_KEY Your OpenRouter API key
OPENROUTER_MODEL Default model override
PROXY_PORT Proxy listen port (default: 9090)

anymodel auto-loads .env from the current directory.

Request Routing

/v1/messages → Routed to your chosen provider
/v1/* → Passed through to api.anthropic.com
/health → Returns proxy status JSON

The proxy sanitizes request bodies: strips betas, metadata, thinking, cache_control, and normalizes tool_choice for cross-provider compatibility.

Setup Guides

Get running in 2 minutes.

Get Your OpenRouter Key

1
Go to openrouter.ai/keys and sign up (Google/GitHub login works)
2
Click "Create Key" — give it any name
3
Copy the key (starts with sk-or-v1-)
4
Save it in your project: 
# Add to .env file
OPENROUTER_API_KEY=sk-or-v1-your-key-here

Free tier: no credit card needed. 29 free models at $0. Add credit ($5+) for paid models and higher rate limits.

Set Up Ollama (Local Models)

1
Install Ollama: 
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows: download from ollama.com
2
Pull a coding model: 
ollama pull qwen3-coder:30b

Other options: llama3, deepseek-r1:32b, qwen3.5

3
Start the proxy + connect: 
# Terminal 1:
npx anymodel proxy ollama --model qwen3-coder:30b

# Terminal 2:
ANTHROPIC_BASE_URL=http://localhost:9090 claude

Fully offline. Nothing leaves your machine. Requires 16GB+ RAM for 30B models, 8GB for smaller ones.

FAQ

Common questions.

Is this free?

anymodel itself is free and open source (MIT). OpenRouter has 29 free models at $0 cost — no credit card needed. Paid models cost per-token through your own OpenRouter account.

Does my API key get stored on anymodel servers?

No. In remote mode, your OpenRouter key passes through the proxy to OpenRouter and is never stored, logged, or cached. In local mode, your key stays entirely on your machine. The proxy is open source — you can verify this yourself.

Do I need a Claude subscription?

For authentication: yes, Claude Code requires a Claude account (free or paid) to start. For model usage: no — anymodel routes the actual AI calls through OpenRouter or Ollama, so you're not using Anthropic API credits.

Which model should I use?

Best free coding model: qwen/qwen3-coder:free — 480B MoE, excellent at code.
Best reasoning: nvidia/nemotron-3-super-120b-a12b:free — chain-of-thought thinking.
Largest context: qwen/qwen3.6-plus:free — 1M tokens.
Best local: qwen3-coder:30b via Ollama.

What's the difference between Claude direct and proxy mode?

Claude direct: npx anymodel claude — just runs Claude Code. No proxy, no OpenRouter key needed.
Proxy mode: Start npx anymodel proxy, then connect with model presets like npx anymodel gpt or manually with ANTHROPIC_BASE_URL=http://localhost:9090 claude. Requires OPENROUTER_API_KEY. Also supports Ollama for fully offline use.

Can I use npx or do I need npm install?

npx anymodel works without installing. If npx fails (some .npmrc configs interfere), install globally instead: npm i -g anymodel. Both work the same way.

What does the proxy actually do?

It sits between Claude Code and OpenRouter. When Claude Code sends a request, the proxy strips Anthropic-specific fields (cache_control, betas, thinking, etc.), normalizes tool_choice, handles retries with exponential backoff, and forwards the cleaned request to OpenRouter. The response streams back unchanged.
